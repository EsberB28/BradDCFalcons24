What is "Container"?

Container and Fluid Container are both very important classes in boostrap. The main difference between the two are:

Container

The class "container" provides a fixed width container. This means that, upon using this class, the result would take up a specific portion of the section they're coding. Its width would also change according to the screens viewing it. For instance, it would appear smaller on a phone than it would on a laptop.

Fluid-container

The .container-fluid class provides a full width container, spanning the entire width of the viewport (width is always 100%). This means that, upon using this class, the result would take up the entire view of the section they're coding. It would also change in size according to the screens viewing it, but maintains its full width on larger screens; smaller screens would see it layed out a bit differently, where it would most likely be lined up.

Conclusion


We can see that the class "fluid-container" takes up more space in the page than the class "container" does because its a full width container. Both work so that any screen could view it, but container handles the job of fitting in the screen better. In the end, both line up in their final form (once on the smallest possible screen size).
